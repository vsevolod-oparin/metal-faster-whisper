# M4.1 Milestone Report — Model Loading & Configuration

**Date:** 2026-03-19
**Status:** PASSED (6/6 tests)

## Summary

Upgraded MWTranscriber from M0 stub to full model loader. On init, it loads the CTranslate2 model, creates a tokenizer from `tokenizer.json`, creates a feature extractor from `preprocessor_config.json`, reads suppression config from `config.json`, and computes all derived constants matching the Python WhisperModel.

## Changes

- **MWTranscriber.h** — expanded API: `featureExtractor`, `tokenizer`, `numLanguages`, derived constants (`inputStride`, `numSamplesPerToken`, `framesPerSecond`, `tokensPerSecond`, `timePrecision`, `maxLength`), `supportedLanguages`, `suppressTokens`, `suppressTokensAtBegin`
- **MWTranscriber.mm** — init loads CT2 model → preprocessor_config.json → MWFeatureExtractor → MWTokenizer → config.json → derived constants → language list. Proper MRC lifecycle (retain in init, release in dealloc). Graceful defaults for missing config files.
- **MWConstants.h** — added `kMWInputStride` (2), `kMWTimePrecision` (0.02), `kMWMaxGenerationLength` (448)
- **MWErrorCode** — added `MWErrorCodeConfigLoadFailed` (300)
