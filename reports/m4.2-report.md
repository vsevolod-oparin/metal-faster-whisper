# M4.2 Milestone Report — Encoding & Language Detection

**Date:** 2026-03-19
**Status:** PASSED (4/4 tests)

## Summary

Added the encoder pipeline (mel → StorageView → CT2 encode) and full language detection with multi-segment support and majority vote fallback.

## Changes

- **MWTranscriber.h** — added `encodeFeatures:nFrames:error:` and `detectLanguageFromAudio:segments:threshold:detectedLanguage:probability:allLanguageProbs:error:`. Added `MWErrorCodeLanguageDetectionFailed`.
- **MWTranscriber.mm** — three additions:
  1. `padOrTrimMel()` static helper — pads/trims row-major mel spectrogram to target frame count
  2. `encodeFeatures:nFrames:error:` — zero-copy StorageView from mel NSData, CT2 encode, float16→float32 conversion
  3. `detectLanguageFromAudio:...` — mel extraction → segment iteration → encode → detect_language → parse `<|en|>` tokens → early stop or majority vote

## Key Finding

**CT2 MPS encoder returns float16 StorageViews.** Calling `output.data<float>()` on float16 triggers `ASSERT_DTYPE`. Fix: `output.to(ctranslate2::DataType::FLOAT32)` before extracting data. The existing M0 `encodeSilenceTestWithError:` never caught this because it only inspected shape, not data.
