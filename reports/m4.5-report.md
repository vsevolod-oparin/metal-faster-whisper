# M4.5 Milestone Report — Segment Splitting by Timestamps

**Date:** 2026-03-19
**Status:** PASSED (5/5 tests)

## Summary

Ported Python's `_split_segments_by_timestamps()`. Parses timestamp tokens from generate() output and splits them into timed `MWSegmentInfo` objects with correct start/end times and seek advancement.

## Changes

- **MWTranscriber.h** — added `MWSegmentInfo` class (seek, startTime, endTime, tokens) and `splitSegmentsByTimestamps:timeOffset:segmentSize:segmentDuration:seek:outSeek:outSingleTimestampEnding:`
- **MWTranscriber.mm** — implemented segment splitting logic:
  - Detects `singleTimestampEnding` (second-to-last < timestampBegin ≤ last)
  - Finds consecutive timestamp indices (both token[i] and token[i-1] ≥ timestampBegin)
  - Splits at consecutive boundaries, computes times from token values × timePrecision (0.02s)
  - Handles no-timestamp case: single segment with full duration
  - Seek update: segmentSize for single endings, lastTimestamp × inputStride otherwise

## Timestamp Token Math

- `timestampBegin` = 50365 (turbo model)
- Token value → seconds: `(token - timestampBegin) × 0.02`
- Seek advancement: `(token - timestampBegin) × inputStride` (in mel frames)
- 30s chunk = 1500 timestamp positions × 0.02s
